Структура проекта:

# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные и модели данных

В этом разделе описаны интерфейсы, используемые в приложении, и классы-модели.

#### Интерфейс товара (IProduct)

Описывает структуру карточки товара, получаемую с сервера.
id: string - уникальный идентификатор;
description: string - описание товара;
image: string - ссылка на изображение;
title: string - название товара;
category: string - категория товара;
price: number | null - цена товара;

#### Интерфейс покупателя (IBuyer)

Описывает данные, которые пользователь вводит при оформлении заказа.
payment: TPayment - способ оплаты;
email: string - электронная почта;
phone: string - мобильный телефон;
address: string - адрес доставки

#### Класс ProductData (Каталог товаров)

Отвечает за хранение списка всех товаров и управление состоянием выбранного товара для просмотра в модальном окне.
Конструктор:
constructor() - инициализирует пустой массив товаров.
Поля:
\_products: IProduct[] - массив всех товаров в магазине.
\_preview: IProduct | null - товар, выбранный для отображения.
Методы:
setProducts(items: IProduct[]): void - сохраняет массив товаров в модель.

getProducts(): IProduct[] - возвращает список всех товаров.

getProduct(id: string): IProduct - находит и возвращает товар по его ID.

setPreview(item: IProduct): void - устанавливает товар для отображения в модальном окне.

getPreview(): IProduct - возвращает текущий выбранный для предпросмотра товар.

#### Класс BasketData (Корзина)

Управляет списком товаров, которые пользователь добавил для покупки, и рассчитывает итоговые показатели.
Конструктор:
constructor() - инициализирует пустой массив выбранных товаров.
Поля:
\_items: IProduct[] - массив товаров в корзине.
Методы:
getItems(): IProduct[] - возвращает список всех товаров в корзине.

addItem(item: IProduct): void - добавляет новый товар в корзину.

removeItem(id: string): void - удаляет товар из корзины по его ID.

clearBasket(): void - полностью очищает корзину.

getTotalPrice(): number - вычисляет и возвращает суммарную стоимость всех товаров.

getCounter(): number - возвращает количество товаров в корзине.

inBasket(id: string): boolean - проверяет, есть ли уже товар с таким ID в корзине.

#### Класс UserData (Данные покупателя)

Отвечает за хранение информации о пользователе и проверку её корректности перед отправкой на сервер.
Конструктор:
constructor() - устанавливает начальные пустые значения для всех полей.
Поля:
payment: TPayment - вид оплаты.

address: string - адрес доставки.

email: string - почта.

phone: string - телефон.

formErrors: Partial<Record<keyof IBuyer, string>> - объект с описанием ошибок валидации.
Методы:
setUserData(field: keyof IBuyer, value: string): void - сохраняет значение в конкретное поле модели.

getUserData(): IBuyer - возвращает объект со всеми данными покупателя.

clearUserData(): void - сбрасывает все поля после оформления заказа.

validateUserData(): boolean - проверяет заполненность полей.

### Слой коммуникации: Класс AppApi

Наследует базовый класс Api, использует его методы для взаимодействия с конкретными эндпоинтами сервера.
Конструктор:
constructor(baseUrl: string, options?: RequestInit) - принимает базовый URL сервера и настройки запросов.
Методы:
getProductList(): Promise<IProduct[]> - получает массив товаров с сервера, подставляя полный путь к картинкам.

orderProducts(order: IOrder): Promise<IOrderResult> - отправляет данные заказа на сервер и возвращает результат.

# Документация классов слоя Представления (View)

## Базовые классы (родители)

### CardBase (родитель для всех карточек)

Назначение: базовый класс для всех типов карточек товара. Содержит общий функционал: установка заголовка, цены, категории (с модификаторами из categoryMap), изображения, обработка кликов.

В конструкторе находит все элементы карточки (.card__title, .card__price, .card__category, .card__image, .card__button) и сохраняет их в поля. Устанавливает слушатель клика на кнопку или на всю карточку.

Методы-сеттеры: title, price (при null показывает "Бесценно"), category (добавляет модификатор), image.

---

### FormBase (родитель для всех форм)

Назначение: базовый класс для всех форм. Содержит общий функционал: управление кнопкой submit (disabled/active), отображение ошибок валидации.

В конструкторе находит форму, кнопку submit и контейнер для ошибок. Устанавливает слушатели на input (вызывает onInputChange) и submit (вызывает onSubmit).

Методы: set valid (включает/выключает кнопку), set errors (показывает ошибки).

---

## Классы карточек (наследники CardBase)

### CardCatalog

Карточка для галереи каталога. Метод render принимает продукт и отображает его. При клике генерирует событие card:select.

### CardPreview

Карточка для модального окна (полная версия). Добавляет поле description (описание товара). Кнопка меняет текст в зависимости от того, есть ли товар в корзине ("В корзину" / "Убрать из корзины").

### CardBasket

Карточка для корзины (компактная версия). Добавляет поле index (номер товара в корзине) и кнопку удаления. При клике на удаление генерируется событие basket:remove.

---

## Классы форм (наследники FormBase)

### OrderForm

Первая форма оформления заказа. Содержит выбор способа оплаты (онлайн/при получении) и поле адреса. Кнопки оплаты используют модификатор button_alt-active для выделения выбранного варианта.

При изменении данных генерирует событие form:change. При сабмите генерирует событие order:submit.

### ContactsForm

Вторая форма оформления заказа. Содержит поля email и телефон. При изменении данных генерирует событие form:change. При сабмите генерирует событие contacts:submit.

---

## Самостоятельные компоненты (без наследования)

### Basket

Компонент корзины. Отображает список товаров, общую сумму и кнопку "Оформить". Кнопка блокируется, если корзина пуста. При клике на "Оформить" генерируется событие order:start.

### Modal

Модальное окно. Управляет видимостью через модификатор modal_active. В конструкторе устанавливает слушатели на кнопку закрытия и на оверлей. Методы: open (показывает окно), close (скрывает и очищает контент). Генерирует события modal:open и modal:close.

### Page

Главная страница. Управляет галереей (список карточек) и счётчиком корзины в хедере. При клике на иконку корзины генерирует событие basket:open.

### Success

Компонент успешного оформления заказа. Показывает сообщение о списании синапсов. Кнопка "За новыми покупками!" закрывает модальное окно.

## События, генерируемые в приложении

### От представлений (View)

card:select - выбор карточки для просмотра (генерирует CardCatalog)

basket:add - добавление товара в корзину (генерирует CardPreview)

basket:remove - удаление товара из корзины (генерирует CardBasket)

basket:open - открытие корзины (генерирует Page)

order:start - начало оформления заказа (генерирует Basket)

order:submit - отправка первой формы (генерирует OrderForm)

contacts:submit - отправка формы контактов (генерирует ContactsForm)

form:change - изменение данных в форме (генерируют OrderForm и ContactsForm)

modal:open - открытие модального окна (генерирует Modal)

modal:close - закрытие модального окна (генерирует Modal)

### От моделей данных (Model)

products:changed - изменение каталога товаров

preview:changed - изменение выбранного товара

basket:changed - изменение содержимого корзины

user:changed - изменение данных покупателя

## Презентер


Презентер реализован в основном файле приложения (без отдельного класса). Он подписывается на все события от моделей и представлений и связывает их между собой.

Основные обработчики:

- products:changed → обновляет галерею на странице
- basket:changed → обновляет счётчик в хедере
- preview:changed → открывает модальное окно с карточкой товара
- card:select → вызывает setPreview в модели ProductData
- basket:open → рендерит корзину и открывает модальное окно
- order:submit → переключает на форму контактов
- contacts:submit → отправляет заказ на сервер

Презентер не генерирует события, только обрабатывает их. Перерисовка компонентов происходит только при получении событий от моделей данных или при открытии модального окна.
